⏰ FASE 2: Corrección de Zona Horaria

Problema Identificado

El sistema registraba cobros realizados después de las 5 PM con fecha del día siguiente porque usaba toISOString() que convierte a UTC en lugar de respetlar la zona horaria local de El Salvador (CST -6).

Ejemplo del Problema

❌ ANTES

Hora Local: 9 abril, 8 PM
toISOString() → UTC
2025-04-10T02:00Z
Resultado: Registra como 10 abril
✅ DESPUÉS

Hora Local: 9 abril, 8 PM
getFechaHoySV()
2025-04-09
Resultado: Registra como 9 abril

Solución Implementada

Creada función global que respeta la zona horaria de El Salvador:

function getFechaHoySV() { const opciones = { year: 'numeric', month: '2-digit', day: '2-digit', timeZone: 'America/El_Salvador' }; const formatter = new Intl.DateTimeFormat('sv-SE', opciones); return formatter.format(new Date()); }
